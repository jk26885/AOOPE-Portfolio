Secure E-Learning Platform (TDD)

Assignment: Design, implement, and test a secure e-learning platform

The e-learning platform is implemented using a layered monolithic architecture, where the application is organised into distinct layers with clearly defined responsibilities. This design promotes maintainability, separation of concerns, and security, while allowing individual modules to evolve independently. The architecture is also scalable because modules are loosely coupled and could be migrated to microservices in the future if required.

Reflection

Week 10 Reflection

Developing the secure e-learning platform provided valuable experience in applying object-oriented design, security principles and Test-Driven Development (TDD) within a realistic software scenario. Rather than focusing solely on functionality, I considered how the system could be maintained, extended and secured as it evolved. This led me to adopt a layered monolithic architecture, separating the application into distinct modules for User Management, Course Management and Enrolment Management, which improved organisation and reduced coupling between components.

The most significant learning experience was applying TDD to the User Management module. Writing tests before implementation initially felt slower because the required behaviours had to be clearly defined before any code could be written. However, this approach helped me identify requirements early and ensured that security and business rules were built into the design from the outset rather than added later. For example, creating tests to verify that passwords were hashed, duplicate usernames were rejected and invalid email addresses were prevented helped establish clear acceptance criteria before implementation began.

The testing phase demonstrated the practical benefits of TDD. With a comprehensive suite of unit tests covering the platform's public methods, I was able to refactor parts of the application with confidence, knowing that existing functionality would be verified automatically. Moving password-handling functionality into shared utilities improved maintainability and reduced code duplication without affecting system behaviour. This reinforced the value of automated testing as a tool for supporting continuous improvement.

Outcome

Unit Outcome

Overall, this assignment strengthened my understanding of how object-oriented architecture, security principles and TDD work together to produce reliable and maintainable software. Designing and testing the e-learning platform demonstrated that investing time in architecture, security and testing early in the development process leads to a more robust and scalable solution. It also increased my confidence in applying these practices to future software engineering projects
