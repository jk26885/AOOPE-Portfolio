Week 7

Tasks Done

  • Reviewed why secure coding matters as software systems become more complex and interconnected.
  • Studied common vulnerabilities in object-oriented systems: injection attacks, buffer overflows, and related risks.
  • Covered core secure coding techniques — input validation, proper authentication, and exception handling.
  • Refactored existing code to eliminate security vulnerabilities: replacing plain-text password handling with salted, iterated hashing, and adding input validation at every entry point.
Reflection

Week 7 Reflection

Unit 7 reinforced the importance of secure coding as a fundamental aspect of software development rather than an afterthought. Through the lecture materials and readings, I developed a deeper understanding of how vulnerabilities can be introduced during development and how secure coding practices help protect applications from security threats. This was particularly relevant to my professional role in IT security, where protecting systems and data is a key responsibility.

A significant learning point was recognising the range of vulnerabilities that can affect object-oriented systems, including injection attacks, insecure input handling, and inadequate authentication mechanisms. The unit highlighted the importance of adopting a security-first mindset throughout the software development lifecycle rather than attempting to add security controls after a system has been built.

Outcome

Unit Outcome

By completing this week's exercises, I was able to:

  • Explain why secure coding is essential in modern, interconnected software systems.
  • Identify common vulnerabilities in object-oriented code, including injection risks and unsafe comparisons.
  • Apply input validation, secure authentication, and structured exception handling as design defaults, not afterthoughts.
  • Refactor existing code to eliminate identified security vulnerabilities.
